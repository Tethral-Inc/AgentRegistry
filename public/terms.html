ACR Terms of Use

Last updated: April 2026 | Tethral, Inc.

What ACR Collects

ACR collects interaction metadata only. Each receipt contains:

FieldDescriptionRisk
agent_idPseudonymous identifierLow
provider_classFramework (anthropic, openai, etc.)Low
target.system_idWhat system was called (e.g., mcp:github)Medium
duration_msHow long the call tookMedium
statusSuccess, failure, timeoutLow
anomaly.detailFree text (operator-provided)High

What We Do NOT Collect

  • No request or response bodies
  • No API keys, credentials, or tokens
  • No user prompts or agent outputs
  • No personally identifiable information
  • No IP addresses
  • No file contents or source code

How Data Is Used

  • Friction analysis: Your own receipts are used to compute what's costing your agent the most. Only you see your individual data.
  • Threat detection: Anomaly signals are correlated across agents. If multiple independent agents report a skill as suspicious, its threat level is elevated.
  • Population baselines: Aggregate statistics (median latency, failure rates) are computed across all agents for benchmarking. These are aggregate — no individual operator data is exposed.

What Is NOT Shared

Individual receipts are never exposed to other operators. Population baselines are aggregate statistics only. Your agent's specific targets, timing, and activity patterns are visible only to you via the friction report.

Anomaly Detail Field

Do not include credentials, API keys, tokens, or sensitive business information in the anomaly.detail field. This field is stored as-is. You are solely responsible for what you include.

Target Redaction

If you don't want the ACR network to know about a particular target system, don't submit receipts for it. ACR cannot retroactively redact target names from submitted receipts.

Data Retention

  • Raw receipts: archived after 90 days, aggregated into daily summaries
  • Daily summaries: retained indefinitely
  • Agent registrations: expire after 90 days of inactivity

Opt Out

Stop submitting receipts at any time by removing the ACR MCP server, skill, or SDK from your agent. No further data will be collected.

Data Deletion

To request deletion of all data associated with an agent_id, contact privacy@tethral.ai with your agent_id.

Changes

We may update these terms. Changes will be reflected on this page with an updated date.
