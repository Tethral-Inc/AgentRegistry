What ACR Does

Interaction Profile Registry

Agents register their composition and log every external interaction. Signals compile into a behavioral profile over time — the unified record of what an agent does and how it behaves.

Friction Lens

Query your profile for where time and tokens are being lost. Top-target breakdown with p95 latencies, chain overhead, retry waste, directional friction, and population baselines.

Lens System

Friction is one lens on the profile. Additional lenses (reliability, quality) are on the roadmap — each a different way of reading the same underlying behavioral signals.

Jeopardy Notifications

If ACR observes anomalies affecting a component in an agent's composition, the agent is notified. We don't track agent owners — we can only notify the agent itself.

Network Observation

Cross-agent, cross-provider anomaly correlation. When signals from multiple agents converge on the same component, the pattern is amplified and propagated.

HIBP for Agents

ACR is a registry and notification layer, not a security check. We record what happens and propagate signals. Like Have I Been Pwned or contact tracing — we don't evaluate, we register.

About the Skill Registry

ACR maintains an observation layer on skills that exist in public registries. We update it continuously. We are not a security check. If we observe that an agent using a particular skill may be in some way jeopardized, we will make an effort to notify the agent itself. Because we do not track the agent's owner, we have no mechanism to notify them beyond the agent's activities.

Agents don't get skills from ACR — we observe skills that already exist in the ecosystem and track behavioral signals tied to them.
